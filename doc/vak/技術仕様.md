# VAK 診断 - 技術仕様

**作成日:** 2026年1月18日  
**最終更新:** 2026年1月18日  

---

## 概要

本ドキュメントは `doc/vak/仕様書.md` から抽出した技術関連の仕様をまとめたものです。運用・開発チーム向けの要点を記載します。

---

## 技術仕様

### フレームワーク
- Next.js 14 (App Router)
- React 18
- TypeScript 5.3

### スタイリング
- Tailwind CSS 3.4
- ユーティリティファーストによるレスポンシブ設計

### データビジュアライゼーション
- Plotly.js 2.29
- レーダーチャートを使用
- クライアントサイドで静的にレンダリング（SSRは無効）

### デプロイ
- GitHub Pages を利用
- 静的エクスポート（`output: 'export'`）
- ベースパス: `/coaching-app/vak/`

---

## パフォーマンス

### 最適化項目
- 静的サイト生成（SSG）を採用
- 画像最適化は無効（GitHub Pages対応のため）
- コードスプリッティングを適用
- 基本的にクライアントサイドで完結する設計

### ページサイズ目安
- First Load JS: 約90 KB
- Route: 約6.5 KB

---

## 運用上の注意点
- GitHub Pages のキャッシュ反映に時間がかかる場合がある（DNSやCDNの反映に注意）
- Plotly の大規模データ描画は避ける（モバイル端末の負荷を考慮）

---

## UI/UX仕様

### デザインコンセプト
- モバイルファースト
- クリーンでシンプル
- イベントの雰囲気に合う洗練されたデザイン

### カラースキーム
- **プライマリカラー**: rgb(99, 110, 250) - 落ち着いた青紫
- **背景**: グラデーション（青→紫）
- **カード**: 白背景 + シャドウ

### レスポンシブ対応
- モバイル: 1カラム、タッチ最適化
- タブレット: 2-3カラム
- PC: 最大幅制限、中央寄せ

### アニメーション
- ボタンホバー: scale(1.05)
- ボタンクリック: scale(0.95)
- ページ遷移: スムーズスクロール


## プライバシー

- **データ保存なし**: すべてクライアントサイドで処理
- **トラッキングなし**: アナリティクスは導入しない想定
- **Cookie不要**: セッション管理を行わない
- **個人情報収集なし**: メールアドレスは `mailto:` のみで送信する設計（サーバ保存なし）

### 想定される機能追加（運用・開発チーム向けメモ）

**完了済み**
- Next.js版の開発
- GitHub Pagesへのデプロイ
- スマホ最適化

**今後の候補**

- オフライン機能（PWA）はキャッシュ戦略とデータ整合性を検討する
- PDFエクスポートはクライアント処理で実現する場合、生成負荷に注意する
